# NestJS Server Monitor App

A RESTful API built with **NestJS** using **TypeORM** + **SQLite** for data persistence, with **Swagger** auto-generated API docs and **Vitest** for testing.

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | NestJS 11 |
| Database | SQLite (better-sqlite3) |
| ORM | TypeORM |
| Validation | class-validator + ValidationPipe |
| API Docs | @nestjs/swagger |
| Testing | Vitest + unplugin-swc |
| Package Manager | pnpm (workspace) |

## Project Structure

```
src/
├── main.ts                          # App entry point (Swagger + ValidationPipe setup)
├── app.module.ts                    # Root module (TypeORM config + imports)
├── app.controller.ts                # Root controller (GET /)
└── users/
    ├── users.module.ts              # Users module (TypeOrmModule.forFeature)
    ├── users.controller.ts          # REST endpoints (CRUD routes)
    ├── users.service.ts             # Business logic + Repository injection
    ├── dto/
    │   ├── create-user.dto.ts       # DTO for creating user (with validation)
    │   └── update-user.dto.ts       # DTO for updating user (PartialType)
    └── entities/
        └── user.entity.ts           # TypeORM entity (@Entity, @Column)
```

## Architecture Flow

```
HTTP Request
    ↓
Controller          — Route handling, parameter extraction (@Get, @Post, @Body, @Param)
    ↓
ValidationPipe      — Validates request body against DTO (class-validator decorators)
    ↓
Service             — Business logic, error handling (NotFoundException)
    ↓
Repository          — Auto-generated by TypeORM (CRUD operations)
    ↓
SQLite Database     — Local file (db.sqlite)
```

## Key Concepts

### 1. Module System

- **AppModule** — Root module, configures TypeORM connection and imports feature modules
- **UsersModule** — Feature module, registers `User` entity via `TypeOrmModule.forFeature([User])`

### 2. Dependency Injection (DI)

```
Controller ←(inject)— Service ←(inject)— Repository
```

- Services are injected into Controllers via constructor: `constructor(private readonly usersService: UsersService)`
- Repositories are injected into Services with a decorator: `@InjectRepository(User)` (required because generics are erased at runtime)
- `private readonly` ensures dependencies cannot be reassigned

### 3. TypeORM

- **Entity** (`@Entity()`) — Maps class to database table
- **Repository** — Auto-generated by TypeORM, provides `find()`, `findOneBy()`, `save()`, `update()`, `delete()`, `remove()`, `create()` methods
- **`autoLoadEntities: true`** — Entities registered via `forFeature()` are automatically loaded
- **`synchronize: true`** — Auto-syncs schema with entities on startup (⚠️ dev only)

### 4. Validation

`ValidationPipe` is configured globally in `main.ts` with:

- **`transform: true`** — Automatically transforms plain objects into DTO class instances
- **`whitelist: true`** — Strips properties not defined in DTO
- **`forbidNonWhitelisted: true`** — Throws error if unknown properties are sent

### 5. Swagger

- Auto-generates API documentation from controllers and DTOs
- **`@nestjs/swagger` CLI plugin** (in `nest-cli.json`) auto-infers schemas from TypeScript types
- Accessible at `http://localhost:3000/api`

## API Endpoints

| Method | Route | Description |
|---|---|---|
| `GET` | `/` | Hello World |
| `POST` | `/users` | Create a new user |
| `GET` | `/users` | Get all users |
| `GET` | `/users/:id` | Get user by ID |
| `PATCH` | `/users/:id` | Update user by ID |
| `DELETE` | `/users/:id` | Delete user by ID |

## Getting Started

```bash
# Install dependencies
pnpm install

# Start dev server (with hot-reload)
pnpm start:dev

# Run tests
pnpm test

# Run tests with coverage
pnpm test:cov
```

- App runs at: `http://localhost:3000`
- Swagger UI: `http://localhost:3000/api`
- Database file: `db.sqlite` (auto-created on first run)
